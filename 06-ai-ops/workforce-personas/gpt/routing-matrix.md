## When to populate

When you upgrade `gpt` (or create a true orchestrator persona like `ceo`),
this matrix becomes important. It tells the persona which other personas to
route specific request types to.

For a leaf persona (one that does the work itself), keep this empty.

For an orchestrator persona, the matrix typically has 5-15 rows mapping
request signals (e.g., "code change", "growth experiment", "support ticket")
to target personas (e.g., `cto`, `cgo`, `cco`) with HITL tier annotations.
